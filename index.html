Panduan Instalasi Git

Sebelum memulai petualangan dengan Git, langkah pertama adalah memasangnya di komputermu. Bagian ini akan memandu Anda melalui proses instalasi Git di berbagai sistem operasi.

Instalasi di Windows

Cara termudah dan paling direkomendasikan adalah menggunakan installer resmi Git untuk Windows:

  1. Unduh installer dari situs resmi: git-scm.com/download/win.
  2. Jalankan file .exe yang sudah diunduh.
  3. Ikuti langkah-langkah pada wizard instalasi. Beberapa poin penting:
    • Pilih editor teks favorit Anda (misalnya, Visual Studio Code, Notepad++).
    • Pastikan opsi "Git from the command line and also from 3rd-party software" terpilih.
    • Untuk opsi lain, pengaturan default biasanya sudah cukup baik.

Setelah instalasi, buka Command Prompt atau PowerShell dan ketik git --version untuk memverifikasi.

Instalasi di macOS

Ada beberapa cara untuk menginstal Git di macOS:

  1. Homebrew (Direkomendasikan): Jika Anda memiliki Homebrew, buka Terminal dan jalankan: brew install git.
  2. Xcode Command Line Tools: Coba jalankan git --version di Terminal. Jika belum ada, sistem mungkin akan menawarkan untuk menginstalnya.
  3. Installer Resmi: Unduh installer dari git-scm.com/download/mac.

Setelah instalasi, buka Terminal dan ketik git --version untuk memverifikasi.

Instalasi di Linux

Instalasi Git di Linux biasanya dilakukan melalui package manager bawaan distro:

  • Distro berbasis Debian/Ubuntu:
    1. Buka Terminal.
    2. Jalankan: sudo apt update
    3. Kemudian: sudo apt install git
  • Distro berbasis Fedora:
    1. Buka Terminal.
    2. Jalankan: sudo dnf install git

Setelah instalasi, buka Terminal dan ketik git --version untuk memverifikasi.

Konfigurasi Awal Setelah Instalasi

Setelah Git terinstal, penting untuk memberitahu Git siapa Anda. Buka Terminal atau Command Prompt dan jalankan:

git config --global user.name "Nama Anda"
git config --global user.email "emailanda@example.com"

Cek konfigurasi: git config --list.

Kenalan Sama Git, Yuk!

Bagian ini akan memperkenalkan Anda pada konsep dasar Git dan mengapa ia menjadi alat yang sangat penting bagi para pengembang.

Apa Itu Git & Version Control?

Git adalah **sistem pencatat super canggih** untuk setiap perubahan pada file proyek Anda. Secara formal, Git adalah *Version Control System* (VCS) yang merekam perubahan file dari waktu ke waktu, memungkinkan Anda kembali ke versi tertentu kapan pun.

Manfaat Utama Menggunakan Git

  • Organisasi Proyek Rapi
  • Kolaborasi Tim Mudah
  • Jaring Pengaman (Commit = Titik Aman)
  • Jejak Perubahan Jelas
  • Eksperimen Aman dengan Branch

Kolaborasi dengan Tim

Jelajahi interaksi dengan repositori online, pahami `pull` vs `fetch`, dan cara menangani `merge conflict`.

Interaksi dengan Repositori Online (Remote)

git clone

Salin repo online ke lokal.

git push

Kirim commit lokal ke online.

git pull

Ambil & gabung update online.

git fetch

Ambil update, belum gabung.

Memahami `git pull` vs `git fetch`

`pull` = `fetch` + `merge`. `fetch` memberi lebih kontrol.

`git fetch`

Unduh "kabar terbaru", tinjau dulu sebelum merge manual.

git fetch + git merge

`git pull`

Langsung unduh DAN gabungkan. Praktis, kurang fleksibel.

git pull

Menangani "Bentrok" Kode (Merge Conflict)

Konflik terjadi saat Git tak bisa otomatis gabung perubahan. Selesaikan manual.

Tips & Kesalahan Umum untuk Pemula

Tips praktis dan kesalahan umum untuk membantu perjalanan Git Anda.

üåü Tips Praktis Sehari-hari

  • Sering commit dengan pesan jelas.
  • Selalu pull sebelum kerja (tim).
  • Gunakan branch untuk tiap tugas.
  • Tulis pesan commit jelas (APA & MENGAPA).
  • Jangan push kode rusak.
  • Komunikasi dengan tim.
  • Manfaatkan git status.

üö´ Kesalahan Umum Pemula & Cara Menghindarinya

  • Lupa git add sebelum commit. Solusi: Cek git status.
  • Kerja di branch salah. Solusi: Cek git branch.
  • Simpan file tak perlu dilacak. Solusi: Pakai .gitignore.
  • Pesan commit tak informatif. Solusi: Jelaskan spesifik.
  • git push --force sembarangan. Solusi: Hindari/koordinasi.
  • Takut merge conflict. Solusi: Pahami & selesaikan tenang.

Perintah Bermanfaat Lainnya

Referensi cepat beberapa perintah Git yang sering digunakan.

Membatalkan commit/perubahan di Staging. Hati-hati, bisa ubah riwayat. Mode: `--soft`, `--mixed`, `--hard`.

Cara aman batal commit. Buat commit baru kebalikan dari commit target. Riwayat utuh.

File khusus untuk daftar file/folder yang diabaikan Git (misal: log, `node_modules`).

Lihat penulis terakhir tiap baris file & ID commit-nya. Berguna untuk melacak perubahan.

Lihat perbedaan antar keadaan repositori (working dir, staging, commit, branch).

  • git diff: Working dir vs Staging.
  • git diff --staged: Staging vs Commit terakhir.
  • git diff <commit1> <commit2>: Antar dua commit.
  • git diff <branch1>..<branch2>: Antar dua branch.
